Quick start

From zero to your first edit. In about 15 minutes.

Five one-time steps. Once you've done them, the toolkit just works — every future change is a conversation with Claude. This guide is for the very first time only.

Time ~15 minutes
What you need Your Mac, a GitHub account, your Claude login
Confidence level Comfortable copy-pasting two commands. No coding required.
1

Get access to the website repo ~1 min · needs a partner

The Magnus website lives in a private GitHub repo at github.com/ssu96ld/magnusconsulting-co-uk. You need to be invited as a collaborator before anything else will work.

What to do

  1. Tell the build team (or the partner who owns the website) your GitHub username — the one you'll be signed in as.
  2. Wait for the invitation email. It comes from noreply@github.com.
  3. Click Accept invitation in the email — this opens GitHub in your browser and confirms access.
  4. Sanity check: visit https://github.com/ssu96ld/magnusconsulting-co-uk. If you see the repo, you're in. If you see a 404 page, the invitation hasn't been sent or accepted yet.

Don't skip this

If you go to the next steps without repo access, the toolkit will keep failing with "authentication" or "not found" errors. They look like a tooling problem but they're an access problem.

2

Install the GitHub CLI ~5 min · one terminal command

The toolkit needs Git credentials on your Mac so it can clone, commit, and open pull requests. The cleanest way to set that up — even if you've never opened Terminal before — is to install Homebrew (a package manager) and then the GitHub CLI in one go.

  1. Open Terminal: hit ⌘ + Space, type Terminal, press Return.
  2. Copy the entire command in the box below. Paste it into Terminal and press Return.
// paste this whole line into Terminal/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" && eval "$(/opt/homebrew/bin/brew shellenv 2>/dev/null || /usr/local/bin/brew shellenv)" && brew install gh && gh auth login

What to expect — three pause points

  • Password: Your Mac login password (the same one you use to install apps). Nothing appears as you type — that's normal. Press Return.
  • Press RETURN/ENTER to continue or any other key to abort: Just press Return.
  • ? What account do you want to log into? Choose GitHub.com. Then HTTPS → Yes (authenticate Git) → Login with a web browser. Copy the one-time code shown, hit Return, your browser opens, paste the code, click Authorize. Back in Terminal, you'll see a green checkmark.

Total time: ~5 minutes. The longest part is Homebrew downloading and compiling — let it run.

Sanity check

After it finishes, type gh auth status and press Return. You should see ✓ Logged in to github.com with your username. If yes, you're done with step 2.

3

Open Claude ~2 min

You need any one of these — whichever you prefer. They all work with the toolkit.

  • Claude Desktop app — the standalone app for Mac. claude.ai/download
  • Claude Cowork — a feature inside Claude Desktop. Same install, just a different mode within the app.
  • Claude Code — the command-line version. Run npm install -g @anthropic-ai/claude-code in Terminal then claude.

Sign in to your Claude account if you haven't already. The plugin works on a paid plan (Pro / Max / Team / Enterprise).

4

Install the Magnus plugin ~1 min

Inside any Claude session, paste these three commands one at a time. Press Return after each.

/plugin marketplace add ssu96ld/magnus-claude-plugins
/plugin install magnus-site
/reload-plugins

What to expect

The first command tells Claude where to find the plugin. The second installs it. The third reloads Claude so the new skills become available. After that, Claude knows how to edit the Magnus website.

If the first command asks "Are you sure?" or similar, say yes — it's just confirming you want to add a third-party marketplace.

5

Get the website onto your Mac ~3 min

Tell Claude — in plain English — to set things up:

Run magnus-setup

What Claude will do

  1. Check if you already have the website folder on your Mac. (You don't — it's your first time.)
  2. Offer to clone it for you. Say yes.
  3. Ask where to put it. Default is fine: a subfolder in your current location, named magnusconsulting-co-uk.
  4. Run npm install to set up the website's dependencies. This takes 30 seconds to a minute.
  5. Tell you the absolute path of the cloned folder, and ask you to start a new Claude session with that folder as the working directory.

Why a new session?

Claude's idea of "what folder am I in" is set when the session starts and doesn't change mid-flow. To use the rest of the toolkit, you start fresh inside the magnus folder. Once there, Claude has everything it needs.

  • In Claude Code: in Terminal, cd into the cloned folder, then run claude.
  • In Claude Desktop / Cowork: start a new session and pick the cloned folder as the project folder.
You're done

That's the whole setup. The next time you want to edit the site, just open Claude.

From here, you ask Claude to do things in plain English — fix a typo, add an article, add a team member, publish the changes. The full vocabulary lives in the user guide:

If you hit a snag

Common first-run issues.

"Authentication failed" or "Repository not found" when Claude tries to clone

Step 1 — repo access — hasn't completed. Either you haven't been invited yet, or you haven't accepted the invitation email. Visit https://github.com/ssu96ld/magnusconsulting-co-uk in a browser to confirm you can see it. If you see a 404, ask the build team to invite you (and check Spam for the invite email).

Terminal says command not found: brew after pasting the install command

Homebrew installed but the shell session doesn't see it yet. Quit Terminal entirely (⌘+Q), open a fresh Terminal, and try gh auth login directly. If that works, you're set — the previous steps already finished.

Claude says it can't find authentication and suggests installing gh

You've already installed gh in step 2 — Claude is just defaulting to that suggestion. Run gh auth status in Terminal to confirm you're still logged in. If yes, the issue is repo access (step 1). If gh auth status says you're logged out, run gh auth login again.

Anything else

Send the build team a screenshot of what Claude said. Most issues are quick fixes once they can see what error came up.
