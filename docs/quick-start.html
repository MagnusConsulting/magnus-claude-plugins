Quick start

From zero to your first edit. In about 15 minutes.

Five one-time steps. Once you've done them, the toolkit just works — every future change is a conversation with Claude. This guide is for the very first time only.

Time ~15 minutes
What you need Your Mac, a GitHub account, your Claude login
Confidence level Comfortable copy-pasting two commands. No coding required.
1

Get access to the website repo ~1 min · needs a partner

The Magnus website lives in a private GitHub repo at github.com/MagnusConsulting/magnusconsulting-co-uk. You need to be invited as a collaborator before anything else will work.

What to do

  1. Tell the build team (or the partner who owns the website) your GitHub username — the one you'll be signed in as.
  2. Wait for the invitation email. It comes from noreply@github.com.
  3. Click Accept invitation in the email — this opens GitHub in your browser and confirms access.
  4. Sanity check: visit https://github.com/MagnusConsulting/magnusconsulting-co-uk. If you see the repo, you're in. If you see a 404 page, the invitation hasn't been sent or accepted yet.

Don't skip this

If you go to the next steps without repo access, the toolkit will keep failing with "authentication" or "not found" errors. They look like a tooling problem but they're an access problem.

2

Install everything from Terminal ~10 min · one paste

One paste in Terminal does the whole machine setup: installs Homebrew (a package manager), the GitHub CLI, Node.js, signs you in to GitHub, clones the Magnus website's staging branch (the live site for admins — CI auto-deploys it to staging.magnusconsulting.co.uk) into ~/Projects/Claude/magnus-website, and installs its dependencies. After this, Claude has nothing to download — it just opens the folder and works.

  1. Open Terminal: hit ⌘ + Space, type Terminal, press Return.
  2. Copy the entire block below. Paste it into Terminal and press Return.
// paste this whole block into Terminal/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" \
  && eval "$(/opt/homebrew/bin/brew shellenv 2>/dev/null || /usr/local/bin/brew shellenv)" \
  && brew install gh node \
  && gh auth login \
  && mkdir -p ~/Projects/Claude \
  && gh repo clone MagnusConsulting/magnusconsulting-co-uk ~/Projects/Claude/magnus-website -- --branch staging \
  && (cd ~/Projects/Claude/magnus-website && npm install)

What to expect — three pause points

  • Password: Your Mac login password (the same one you use to install apps). Nothing appears as you type — that's normal. Press Return.
  • Press RETURN/ENTER to continue or any other key to abort: Just press Return.
  • ? What account do you want to log into? Choose GitHub.com. Then HTTPS → Yes (authenticate Git) → Login with a web browser. Copy the one-time code shown, hit Return, your browser opens, paste the code, click Authorize. Back in Terminal, you'll see a green checkmark — and the rest of the script continues automatically.

Total time: ~10 minutes. Homebrew downloading and compiling is the longest part, then npm install at the end takes 30–60 seconds. Once Terminal returns you to a normal prompt with no red error text, you're done.

Sanity check

Type ls ~/Projects/Claude/magnus-website and press Return. You should see folders like src, docs, public, and a package.json file. If yes, the website is on your Mac and ready.

Why ~/Projects/Claude/ and not Documents?

On many work Macs, Documents is synced through OneDrive, which silently replaces files with symlinks and breaks git and npm. ~/Projects/Claude/ stays off OneDrive and is also the path the toolkit's helper looks in by default — so every doc and skill assumes it.

3

Open the project in Claude ~2 min

You need any one of these — whichever you prefer. They all work with the toolkit.

  • Claude Desktop app — the standalone app for Mac. claude.ai/download
  • Claude Cowork — a feature inside Claude Desktop. Same install, just a different mode within the app.
  • Claude Code — the command-line version. Run npm install -g @anthropic-ai/claude-code in Terminal then claude.

Sign in to your Claude account if you haven't already. The plugin works on a paid plan (Pro / Max / Team / Enterprise).

Point Claude at the cloned folder

Step 2 cloned the website to ~/Projects/Claude/magnus-website. Open Claude there so every skill resolves paths correctly.

  • Claude Desktop / Cowork: click + New Project, name it magnus-website (with a hyphen), and choose ~/Projects/Claude/magnus-website as the folder. The folder already exists with the website inside it — Claude just adopts it.
  • Claude Code CLI: in Terminal, run cd ~/Projects/Claude/magnus-website && claude.
4

Install the Magnus plugin ~1 min

Add the Magnus marketplace through Claude's UI, then install the plugin from it.

  1. In Claude (Cowork or Code), click Customise in the top-left menu.
  2. Click the + button, choose Create plugin, then Add marketplace.
  3. In the dialog that appears, paste the GitHub path below and confirm:
MagnusConsulting/magnus-claude-plugins
  1. Click Personal in the same menu — you'll see a new plugin called magnus-site. Click it, then install.

What to expect

Claude may show a security or trust warning when you add the marketplace or install the plugin — accept it. Once installed, Claude knows how to edit the Magnus website.

5

Confirm it all works ~1 min

In the chat for your magnus-website project, type:

Run magnus-setup

What you should see

Claude will check the project folder for the three markers it needs and respond with something like:

"You're in the magnus repo at /Users/<you>/Projects/Claude/magnus-website. Everything is ready — go ahead and use any of the magnus-site skills."

If you see that, the toolkit is wired up correctly. You're done.

If Claude says it can't find the magnus repo

The project was opened against the wrong folder. Close the chat and re-create the Claude Project pointed at ~/Projects/Claude/magnus-website (Claude Desktop / Cowork), or run cd ~/Projects/Claude/magnus-website && claude (Claude Code). Then try Step 5 again.

You're done

That's the whole setup. The next time you want to edit the site, just open Claude.

From here, you ask Claude to do things in plain English — fix a typo, add an article, add a team member, publish the changes. The full vocabulary lives in the user guide:

If you hit a snag

Common first-run issues.

"Authentication failed" or "Repository not found" during the gh repo clone step

Step 1 — repo access — hasn't completed. Either you haven't been invited yet, or you haven't accepted the invitation email. Visit https://github.com/MagnusConsulting/magnusconsulting-co-uk in a browser to confirm you can see it. If you see a 404, ask the build team to invite you (and check Spam for the invite email). Once access is fixed, re-run just the clone + install part: gh repo clone MagnusConsulting/magnusconsulting-co-uk ~/Projects/Claude/magnus-website -- --branch staging && (cd ~/Projects/Claude/magnus-website && npm install).

The paste stopped partway with a red error

The chained commands run with &&, so the first failure halts the rest. Read the last red line in Terminal — it almost always names what failed (Homebrew, gh auth, the clone, or npm install). Fix that one thing, then re-run from where it stopped. The script is safe to re-run end-to-end: already-installed pieces are skipped.

Terminal says command not found: brew

Homebrew installed but the shell session doesn't see it yet. Quit Terminal entirely (⌘+Q), open a fresh Terminal, and paste the same block again — already-installed pieces will skip and the rest will run.

Claude says "This folder isn't the magnus repo"

The Claude Project is pointed at the wrong folder. Close the chat, delete the project (or create a new one), and point it at ~/Projects/Claude/magnus-website. In Claude Code, run cd ~/Projects/Claude/magnus-website && claude instead.

Anything else

Send the build team a screenshot of what Claude said. Most issues are quick fixes once they can see what error came up.
